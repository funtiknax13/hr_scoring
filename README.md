# HR Scoring

Внутренний инструмент отдела АБП. Две задачи:

1. **LLM-скоринг резюме** — ранжирование кандидатов под вакансию с обоснованием
2. **Сбор рыночных данных** — вакансии и зарплатные вилки с HH и SuperJob

---

## Быстрый старт

```bash
pip install -r requirements.txt
cp .env.example .env   # заполнить ключи
```

---

## Скрипты

### `score.py` — скоринг резюме

Три фазы: рубрика из вакансии → профиль кандидата → оценка по критериям.
Итоговый балл считается в Python, не доверяется LLM.

**Структура данных:**
```
data/
  <файл вакансии>      # один файл любого формата
  resumes/
    <резюме 1>
    <резюме 2>
    ...
```

**Поддерживаемые форматы:** `.md`, `.txt`, `.pdf`, `.docx`, `.html`

```bash
python score.py
```

**Опции:**
```
--data-dir    папка с вакансией (по умолчанию: data)
--vacancy     путь к файлу вакансии (если несколько в папке)
--resumes-dir папка с резюме (по умолчанию: data/resumes)
--model       модель DeepSeek (по умолчанию: deepseek-chat)
--no-cache    не использовать кэш LLM-ответов
--out         путь к JSON-отчёту (по умолчанию: results.json)
```

Результат — таблица ранжирования в консоли и подробный `results.json`.

### `hh_fetch.py` — вакансии с HeadHunter

> **Статус:** ожидает одобрения заявки на [dev.hh.ru](https://dev.hh.ru).
> Публичный API для соискателей закрыт с декабря 2025, требуется OAuth.

**Первичная настройка (один раз):**

1. Зарегистрируй приложение на [dev.hh.ru](https://dev.hh.ru)
   - тип: «Только сотрудники работодателя»
   - `redirect_uri = http://localhost:8765`
2. Добавь в `.env`:
   ```
   HH_CLIENT_ID=...
   HH_CLIENT_SECRET=...
   HH_USER_AGENT=HR-Scoring-Tool/1.0 (your@email.com)
   ```
3. Получи токен:
   ```bash
   python hh_fetch.py --auth
   ```
   Скрипт напечатает ссылку → открой в браузере → войди в HH →
   скопируй URL из адресной строки (с `?code=...`) → вставь в терминал.
   Токен сохранится в `.env` автоматически.

**Поиск вакансий:**
```bash
python hh_fetch.py
```

Скрипт спросит название вакансии, регион и количество страниц.
Результат — CSV-файл в `data/` с префиксом `hh_`.

### `sj_fetch.py` — вакансии с SuperJob

OAuth не нужен — только API-ключ.

**Настройка:**
```
SJ_API_KEY=v3.r.1...   # в .env
```

```bash
python sj_fetch.py
```

Скрипт спросит название вакансии и город.
Результат — CSV-файл в `data/` с префиксом `sj_`.

## Формат CSV (HH и SuperJob)

| id | название | зарплата_от | зарплата_до | валюта | локация |
|----|----------|-------------|-------------|--------|---------|

Кодировка: UTF-8, разделитель: `;`

---

## Переменные окружения

| Переменная | Описание |
|---|---|
| `DEEPSEEK_API_KEY` | Ключ DeepSeek API (обязателен для скоринга) |
| `DEEPSEEK_BASE_URL` | URL API, по умолчанию `https://api.deepseek.com` |
| `SJ_API_KEY` | Ключ SuperJob API |
| `HH_CLIENT_ID` | Client ID приложения HH |
| `HH_CLIENT_SECRET` | Client Secret приложения HH |
| `HH_TOKEN` | OAuth-токен HH (заполняется через `--auth`) |
| `HH_USER_AGENT` | Идентификатор приложения для HH API |
